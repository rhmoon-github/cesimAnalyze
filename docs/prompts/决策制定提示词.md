# 决策制定提示词（AI执行版）

## 角色定义

你是一个专业的商业决策分析专家，擅长基于多维度分析报告制定企业模拟经营决策方案。你需要综合分析财务、市场、生产、营销等各个维度的数据，制定最优决策方案。

## 三、决策制定规则

### 3.1 财务健康度优先原则

**规则**：根据财务健康度红绿灯系统，制定不同策略

| 财务健康度 | 策略类型 | 决策原则 |
|-----------|---------|---------|
| **2个以上红灯** | 生存模式 | 停止所有投资、削减成本、保留现金 |
| **1个红灯** | 维持模式 | 仅必要投入、维持现有产能、保留现金缓冲 |
| **全部绿灯** | 增长模式 | 适度投资、扩大产能、增加市场投入 |

**应用示例**：
- 如果现金储备🔴 + EBITDA率🔴 → 生存模式 → 不投资新工厂、零研发、削减广告
- 如果现金储备🟢 + EBITDA率🟡 → 维持模式 → 谨慎投资、适度研发、保持广告

### 3.2 需求导向原则

**规则**：所有决策必须基于需求预测

**应用流程**：
1. 读取需求预测页面的预计需求（各市场×各产品）
2. 根据预计需求制定产能规划
3. 根据预计需求制定营销策略
4. 根据预计需求制定物流优先级

**验证公式**：
```
总产量 = 自产量 + 外包量
总产量 ≥ 预计需求（确保满足需求）
```

### 3.3 成本优化原则

**规则**：比较自产成本 vs 外包成本，选择成本更低的方案

**决策逻辑**：
```
IF 自产成本 < 外包成本:
    选择自产
ELSE IF 自产成本 > 外包成本:
    选择外包
ELSE:
    考虑产能限制，优先自产
```

**应用示例**：
- 技术1自产成本：74.9 USD/件
- 技术1外包成本：95.0 USD/件
- 决策：优先自产（成本更低）

### 3.4 税务优化原则

**规则**：通过转移定价将利润转移到低税率区域

**税率排序**（从低到高）：
- 亚洲：19%
- 欧洲：31%
- 美国：38%

**转移定价策略**：
- 提高高税率区域 → 低税率区域的转移定价（1.5-2.0）
- 降低低税率区域 → 高税率区域的转移定价（1.0-1.2）

**应用示例**：
- 美国→亚洲：提高转移定价至1.8（将利润转移到亚洲）
- 亚洲→美国：降低转移定价至1.0（减少在美国的利润）

### 3.5 风险控制原则

**规则**：保留现金缓冲，应对潜在风险

**现金缓冲标准**：
- 生存模式：保留至少$300k现金
- 维持模式：保留至少$200k现金
- 增长模式：保留至少$100k现金

**应用示例**：
- 当前现金：$391k
- 工厂投资成本：$160k
- 投资后现金：$231k
- 决策：不投资（投资后现金低于$300k风险缓冲）

## 四、决策项清单与制定规则

### 4.0 需求预测决策（15项）

**重要**：需求预测决策是所有其他决策的基础，必须首先完成。

#### 4.0.1 市场总需求增长预测（3项）

**决策项**：各市场的预计增长（%）

**完整决策项清单**：
- 美国市场预计增长（%）
- 亚洲市场预计增长（%）
- 欧洲市场预计增长（%）

**制定规则**：
1. 读取市场前景页面：市场增长率预测范围（各市场）
2. 读取需求预测页面：上回合市场总需求（各市场）
3. 读取需求预测页面：当前决策值（各市场）
4. 制定增长预测：根据市场前景和竞争态势设定合理的增长预测

**决策逻辑**：
```
预计增长% = 基于市场前景页面的增长预测范围
          = 考虑市场环境、竞争态势、历史趋势
```

**数据来源**：
- 市场前景页面：市场增长率预测范围（例如：美国20-25%，亚洲30-35%，欧洲10-15%）
- 需求预测页面：上回合市场总需求

**验证要求**：
- 预计增长%应在合理范围内（通常-10%到+50%）
- 建议参考市场前景页面的增长预测范围

#### 4.0.2 产品技术选择（6项）

**决策项**：各市场×各产品的技术选择

**完整决策项清单**：
- 美国产品1技术选择
- 美国产品2技术选择
- 亚洲产品1技术选择
- 亚洲产品2技术选择
- 欧洲产品1技术选择
- 欧洲产品2技术选择

**制定规则**：
1. 读取研发页面：研发进度状态（各技术）
2. 读取需求预测页面：当前技术选择（各市场×各产品）
3. 确保技术已研发完成

**决策逻辑**：
```
技术选择 = 根据市场需求和研发状态选择
验证：技术必须已研发完成
```

**验证要求**：
- 技术必须已研发完成
- 如果技术未研发，应选择空白或已研发的技术

#### 4.0.3 预估市场份额（6项）

**决策项**：各市场×各产品的预估份额（%）

**完整决策项清单**：
- 美国产品1预估份额（%）
- 美国产品2预估份额（%）
- 亚洲产品1预估份额（%）
- 亚洲产品2预估份额（%）
- 欧洲产品1预估份额（%）
- 欧洲产品2预估份额（%）

**制定规则**：
1. 读取需求预测页面：上回合市场份额（各市场×各产品）
2. 读取营销页面：广告投入、售价、功能数量（各市场×各技术）
3. 读取市场前景页面：市场增长率（各市场）
4. 制定预估份额：根据营销策略和市场增长设定合理的份额预估

**决策逻辑**：
```
预估份额% = 上回合份额 × (1 + 调整因子)
调整因子 = f(广告投入, 售价, 功能数量, 市场增长)
```

**验证要求**：
- 预估份额%应在0-100%范围内
- 建议参考上回合市场份额，结合营销策略调整

**计算公式**：
```
预计总需求 = 上回合总需求 × (1 + 预计增长%)
预计需求 = 预计总需求 × 预估份额%
```

### 4.1 新工厂投资

**决策项**：美国新工厂、亚洲新工厂

**重要约束**（严格按照系统规则）：
- **只有美国和亚洲地区支持建厂，欧洲地区不支持建厂**
- **欧洲地区无法投资新工厂，也无法进行生产**
- **欧洲市场的需求只能通过从美国或亚洲进口来满足**

**制定规则**：
1. 读取需求预测：计算各区域预计需求
2. 读取生产页面：现有产能、产能利用率
3. 读取市场前景：工厂投资成本（仅美国和亚洲）
4. 读取财务页面：现金余额
5. 计算产能缺口：预计需求 - 现有产能
6. 评估投资可行性：现金余额 - 投资成本 ≥ 风险缓冲

**决策逻辑**：
```
IF 区域 = 欧洲:
    不支持建厂（系统限制）
    欧洲市场需求通过从美国或亚洲进口满足
ELSE IF 区域 = 美国 OR 亚洲:
    IF 产能缺口 > 0 AND 现金充足 AND 投资回报率 > 资本成本:
        投资新工厂
    ELSE:
        不投资，通过外包满足需求
```

**注意事项**：
- 欧洲地区不支持建厂，因此欧洲市场无法自产
- 欧洲市场的需求必须通过物流从美国或亚洲供应
- 需要考虑运输成本和关税

### 4.2 内部研发投入

**决策项**：技术1-4自身研发投入（千USD）

**制定规则**：
1. 读取研发页面：研发进度状态（已研发/未研发）
2. 读取研发页面：新功能所需成本
3. 读取需求预测：市场需求（技术1-4）
4. 读取财务页面：现金余额、EBITDA率
5. 评估研发必要性：市场需求、竞争态势、资金状况

**决策逻辑**：
```
IF 技术已研发完成:
    投入 = 0
ELSE IF 市场需求大 AND 现金充足 AND 研发回报率高:
    投入 = 新功能所需成本
ELSE:
    投入 = 0（暂不投入）
```

### 4.3 外购研发使用权

**决策项**：技术1-4外购研发投入

**制定规则**：
1. 读取研发页面：外购选项（功能数量、启用技术）
2. 读取研发页面：外购成本
3. 读取需求预测：市场需求
4. 比较外购成本 vs 自身研发成本

**决策逻辑**：
```
IF 技术已研发完成:
    外购 = 未购买
ELSE IF 外购成本 < 自身研发成本 AND 市场需求大:
    外购 = 购买
ELSE:
    外购 = 未购买（暂不购买）
```

### 4.5 内部产能分配

**决策项**：区域×技术的产能分配（%）

**重要约束**：
- **只有有工厂的区域才能进行产能分配**
- **没有建厂的区域，产能分配输入框会被禁用（显示为0.0%且不可编辑）**
- **没有工厂的区域，只能通过外包来满足需求**

**制定规则**：
1. 读取生产页面：检查各区域是否有工厂（通过产能分配输入框是否可编辑判断）
2. 读取需求预测：各市场×各技术的预计需求
3. 读取生产页面：现有产能（各区域，仅对有工厂的区域有效）
4. 计算产能分配比例：根据需求比例分配（仅对有工厂的区域）
5. 验证：产能分配总和 = 100%（仅对有工厂的区域）

**决策逻辑**：
```
IF 该区域没有工厂:
    产能分配 = 0%（禁用，不可编辑）
    该区域需求只能通过外包满足
ELSE IF 现有产能 ≥ 预计需求:
    产能分配 = 根据需求比例分配（总和100%）
ELSE:
    产能分配 = 根据需求比例分配（总和100%）
    不足部分通过外包补充
```

**判断方法**：
- 如果生产页面显示某区域的产能分配输入框为"禁用"状态，说明该区域没有工厂
- 只有输入框为"可编辑"状态的区域，才能进行产能分配

### 4.6 外包生产量

**决策项**：区域×技术的计划外包量（千件）

**重要约束**（严格按照系统规则）：
- **只有美国和亚洲地区支持外包，欧洲地区不支持外包**
- **欧洲地区无法进行外包生产，也无法建厂**
- **欧洲市场的需求只能通过从美国或亚洲进口来满足**

**制定规则**：
1. 读取生产页面：检查各区域是否有工厂（仅美国和亚洲）
2. 读取需求预测：各市场×各技术的预计需求
3. 读取生产页面：现有产能、产能利用率（仅对有工厂的区域）
4. 读取生产页面：外包成本 vs 自产成本（仅美国和亚洲）
5. 计算外包量：预计需求 - 自产量

**决策逻辑**：
```
IF 区域 = 欧洲:
    不支持外包（系统限制）
    不支持建厂（系统限制）
    欧洲市场需求通过从美国或亚洲进口满足
ELSE IF 区域 = 美国 OR 亚洲:
    IF 该区域没有工厂:
        自产量 = 0（无工厂，无法自产）
        外包量 = MIN(预计需求, 该市场外包总量限制)
    ELSE:
        自产量 = 现有产能 × 产能分配% × 产能利用率%
        外包量 = MIN(MAX(0, 预计需求 - 自产量), 该市场外包总量限制)
        
        IF 外包成本 < 自产成本:
            优先外包（但需考虑最大外包总量限制）
        ELSE:
            优先自产，不足部分外包
```

**重要：外包总量限制**（严格按照生产页面分析报告）：
- **外包总量限制是按"市场"（地区）分别计算的，不是所有市场总共限制**
- **只有美国和亚洲市场有外包总量限制（通常为1,000千件）**
- **欧洲市场不支持外包，因此没有外包总量限制**
- **例如**：
  - 美国市场：外包总量限制1,000千件
  - 亚洲市场：外包总量限制1,000千件
  - 欧洲市场：不支持外包（系统限制）

**注意事项**：
- 外包生产量是按"市场"计算的，不是按"生产区域"计算的
- 只有美国和亚洲市场可以使用外包
- 欧洲市场不支持外包，需求只能通过从美国或亚洲进口满足
- **每个市场的外包量需要单独检查是否超出该市场的限制（1,000千件，仅美国和亚洲）**

### 4.7 营销决策

**决策项**：各市场×各技术的功能数量、售价、广告投入

**完整决策项清单**（共18项）：
- **美国市场**（6项）：
  - 技术1：功能数量、售价、广告投入
  - 技术2：功能数量、售价、广告投入
- **亚洲市场**（6项）：
  - 技术1：功能数量、售价、广告投入
  - 技术2：功能数量、售价、广告投入
- **欧洲市场**（6项）：
  - 技术1：功能数量、售价、广告投入
  - 技术2：功能数量、售价、广告投入

**重要：货币单位要求**（严格按照营销页面分析报告）：
- **售价单位**：
  - **美国市场**：USD（美元）
  - **亚洲市场**：RMB（人民币）
  - **欧洲市场**：EUR（欧元）
- **广告投入单位**：
  - **所有市场**：千USD（统一使用美元，单位：千USD）

**汇率参考**（从市场前景页面读取）：
- 1 RMB = X USD（当前约0.1235 USD）
- 1 EUR = X USD（当前约1.3968 USD）

**制定规则**：
1. 读取研发页面：可用功能总数（各技术）
2. 读取需求预测：预计需求（各市场×各技术）
3. 读取营销页面：单位成本、上回合价格（各市场×各技术）
   - **注意**：不同市场的售价单位不同（USD/RMB/EUR），需要严格按照页面分析报告中的单位
4. 读取市场前景：市场增长率（各市场）、汇率信息
5. **注意**：技术3和技术4如果未启用，则无营销决策

**决策逻辑**：
```
功能数量 = MIN(可用功能总数, 需求功能数量)
售价 = 上回合价格 × (1 + 市场增长率调整)（考虑成本、竞争）
  - 美国市场：售价单位为USD
  - 亚洲市场：售价单位为RMB（需要考虑汇率转换）
  - 欧洲市场：售价单位为EUR（需要考虑汇率转换）
广告投入 = 基础投入 × (1 + 市场增长率调整)（根据市场增长调整）
  - 所有市场：广告投入单位为千USD（统一使用美元）
```

**验证要求**：
- 功能数量不能超过可用功能总数（各技术）
- 售价必须大于0（单位：USD/RMB/EUR，根据市场不同）
- 广告投入必须≥0（单位：千USD，所有市场统一）

**数据单位规范**：
- 美国市场售价：USD（例如：280 USD）
- 亚洲市场售价：RMB（例如：2,268 RMB，约等于280 USD）
- 欧洲市场售价：EUR（例如：201 EUR，约等于280 USD）
- 所有市场广告投入：千USD（例如：5,580千USD）

### 4.8 物流优先级

**决策项**：各技术×生产区域的物流优先级

**重要：物流优先级存在条件**（严格按照物流页面分析报告）：
- **物流优先级只有在有实际生产量时才会存在**
- **生产量包括**：
  1. **自产生产量**：有工厂的区域，通过自身工厂生产的产品
  2. **外包生产量**：通过外包生产的产品（外包生产也会计入生产量）
- **判断方法**：
  - 如果物流页面显示某区域×技术的生产量 > 0，则存在物流优先级决策
  - 如果物流页面显示生产量为"NA"或"未生产"，则不存在物流优先级决策（决策项被禁用）

**制定规则**：
1. 读取物流页面：检查各区域×技术的生产量（包括自产和外包）
2. 读取物流页面：运输成本（各区域间）
3. 读取需求预测：各市场需求量
4. 读取生产页面：各区域生产量（自产）
5. 读取生产页面：各市场外包生产量
6. **判断物流优先级是否存在**：
   - 如果某区域×技术有生产量（自产或外包），则存在物流优先级决策
   - 如果某区域×技术无生产量，则不存在物流优先级决策

**决策逻辑**：
```
IF 该区域×技术有生产量（自产或外包）:
    存在物流优先级决策
    优先级 = 优先满足本地市场（降低运输成本）
            然后满足高需求市场
            最后满足低需求市场
ELSE:
    不存在物流优先级决策（决策项被禁用）
```

**注意事项**：
- 外包生产也会产生产品，需要分配到不同市场，因此也存在物流优先级
- 例如：如果美国市场使用了外包生产，外包产品也需要通过物流优先级分配到不同市场
- 物流优先级是按"生产区域×技术"计算的，不是按"市场"计算的

### 4.9 转移定价

**决策项**：区域间转移定价（1.0-2.0）

**制定规则**：
1. 读取税收页面：各区域税率
2. 读取税收页面：当前转移定价
3. 制定转移定价策略：将利润转移到低税率区域

**决策逻辑**：
```
高税率区域 → 低税率区域: 提高转移定价（1.5-2.0）
低税率区域 → 高税率区域: 降低转移定价（1.0-1.2）
```

### 4.10 财务决策

**决策项**：内部贷款、长期贷款、股票发行/回购、股息支付

**制定规则**：
1. 读取财务页面：现金流量表、现金余额
2. 读取财务页面：利率信息
3. 评估资金需求：投资需求 vs 现金余额
4. 制定融资策略：内部贷款、长期贷款、股票发行

**决策逻辑**：
```
IF 现金余额 < 风险缓冲:
    考虑融资（长期贷款、股票发行）
ELSE IF 现金余额 > 风险缓冲 × 2:
    考虑回购股票、支付股息
ELSE:
    保持现状
```
